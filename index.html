② パラメータ

💡 各パラメータの役割

Ch2 状態判定閾値 [V]
Ch2 の電圧を見て「充電中 / 待機 / 放電中」の 3 状態に分類する境界です。 |Ch2| > 閾値 なら充電(+) または放電(-)、それ以下は待機と判定。
通常は 0.1 のままで OK。 このデータは Ch2=±0.2V なので 0.1 が最適値です。
ヘッダスキップ行数
CSV 先頭の見出し行数。T&D Graph 標準なら「Date/Time行 × 2 + 単位行」で 3 行。
T&D Graph の出力なら 3 のままで OK。
📌 検出ロジック (v3 電圧変化ベース)：各サイクルで
🔵 青 = 放電期間 (Ch2<-閾値) の Ch1 最小値 の点 (=底を打った瞬間)
🔴 赤 = 充電期間 (Ch2>+閾値) の Ch1 最大値 の点 (=ピークの瞬間)
🟡 黄 = Ch2 が 0→-0.2V に切り替わる 1 ステップ前 (放電開始直前)
🟢 緑 = Ch2 が 0→-0.2V に切り替わる 最初の点 (放電開始の瞬間)
🎯 丸の位置 (取得箇所) を動かしたい場合
青/赤 を左右に動かしたい アルゴリズム上、青/赤は最小/最大値の点で自動決定されるため、 パラメータで直接動かせません。 → 下の検証グラフで縦線をドラッグして手動調整してください。
黄 を左右に動かしたい 黄は「放電開始直前の点」(緑の 1 ステップ前) なので緑と連動します。 緑の縦線をドラッグすると黄も連動移動します。
緑 を左右に動かしたい 緑の縦線をドラッグしてください。そこが「新しい放電開始点」になり、 黄もその 1 ステップ前に自動移動します。
閾値で動かす 閾値を小さくすると充電・放電期間が広く認識 → 青/赤がより端で取得される可能性
閾値を大きくすると期間が狭く認識 → より中央の値で取得される
